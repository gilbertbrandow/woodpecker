# Decoy pipeline uses Lichess cloud eval API, not the bulk eval dataset

The authoritative source for Lichess engine evaluations is the bulk eval dataset (`database.lichess.org/#evals`), a multi-hundred-GB zstandard-compressed JSONL file. The decoy pipeline instead calls the Lichess cloud eval API (`lichess.org/api/cloud-eval?fen=<FEN>&multiPv=5`) on demand, one FEN at a time. This is possible because we walk master PGN games first and only look up FENs that appear in those games — master game positions are reliably present in the cloud eval cache at depth ≥ 20. The bulk dataset approach was rejected because it requires downloading and indexing hundreds of GB locally before any filtering can happen, adding substantial operational cost for no quality benefit given the high cache hit rate for master game positions. If coverage gaps become a problem in practice, switching to the bulk dataset is the fallback.
